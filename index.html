第 1 节：RAG 基础认知 —— 产品经理必懂的 RAG 核心概念 ▾

核心目标：搞懂 RAG 是什么、解决什么问题、和纯大模型的产品差异，建立产品视角的 RAG 认知框架

  • 从产品痛点切入：纯大模型产品的 3 大核心问题（知识过时、幻觉、专业领域回答不准），结合行政智能助手案例说明
  • RAG 的定义：检索增强生成（Retrieval-Augmented Generation）产品视角的解读 ——"先找答案，再写答案"
  • RAG 的核心价值：产品层面的 3 个核心作用（解决知识过时、降低幻觉、提升专业领域回答准确性）
  • RAG 与纯大模型、传统知识库的产品差异（表格对比，聚焦回答灵活性、知识准确性、更新成本、开发成本）
  • RAG 的适用边界：哪些场景不适合做 RAG（如无固定知识库、开放性创意生成）
  • 小练习：判断 3 个业务场景是否适合 RAG
第 2 节：RAG 核心原理 —— 产品经理只需懂这些（避开纯技术） ▾

核心目标：理解 RAG 的 4 大核心环节，知道每个环节的产品影响点，能和开发/算法高效沟通

RAG 的 4 大核心环节（产品视角拆解）

  • 知识库构建：把非结构化文档变成大模型能"检索"的格式，产品关注文档解析精度、知识库更新方式、权限管理
  • 检索：用户提问后，从知识库中找匹配的"上下文"，产品关注检索精度、检索速度、多关键词匹配能力
  • 提示词拼接：把"检索到的上下文 + 用户问题 + 指令"拼接成提示词发给大模型，产品关注提示词结构、上下文筛选规则
  • 生成：大模型基于拼接后的提示词输出答案，产品关注生成范围、答案贴合度、格式规范

核心组件（产品价值解读）

  • 向量数据库：影响"检索快慢/准度"
  • 嵌入模型：影响"文档和问题的匹配精度"
第 3 节：RAG 产品设计全流程 —— 产品经理的核心工作 ▾

核心目标：掌握从"需求分析"到"产品落地"的 RAG 产品设计全流程，能输出核心设计方案

第一步：需求分析与场景适配

  • 核心判断标准：是否有结构化/非结构化的固定知识库？是否需要基于知识库的精准问答？是否需要频繁更新知识？
  • 需求拆解要点：明确问答边界、答案格式要求、知识库更新频率

第二步：知识库产品设计

  • 知识库类型设计：结构化/非结构化/混合
  • 核心功能设计：文档上传、文档解析、知识更新、知识分册/标签、权限管理

第三步：检索策略产品设计

  • 核心检索方式：关键词检索、向量检索（前者快，后者准，建议混合）
  • 优化点：检索结果数量限制、相似度阈值设置、多轮检索

第四步：生成环节产品设计

  • 答案格式规范：结构化/标注答案来源
  • 兜底逻辑设计：检索不到匹配内容时的回复策略

第五步：产品架构与交互设计

  • 核心架构：用户端→问答模块→检索模块→知识库→大模型→生成结果→用户端
  • 交互设计：问答界面、知识库管理界面
第 4 节：RAG 提示词撰写 —— 产品经理专属方法论（核心重点） ▾

核心目标：掌握 RAG 提示词的核心逻辑，能撰写/设计不同场景的 RAG 提示词模板

  • RAG 提示词与普通大模型提示词的核心差异：必须绑定"检索上下文"
  • 产品经理撰写 RAG 提示词的核心原则（4 个）
  • RAG 提示词的通用结构（可直接复用）
  • 不同场景的 RAG 提示词模板设计
  • 产品侧的提示词优化技巧（无需改代码）
  • 实操练习：为行政智能助手撰写基础问答提示词模板
第 5 节：RAG 落地常见问题 —— 产品侧优化策略 ▾

核心目标：掌握 RAG 落地的 5 大常见问题，能从产品角度提出可落地的优化方案

  • 检索不准 —— 知识标签/切分、检索阈值调整、关键词补充
  • 仍有幻觉 —— 强化提示词指令、限制生成范围、标注答案来源
  • 答案冗长/不贴合 —— 提示词指定答案格式、上下文筛选
  • 知识库更新不及时 —— 自动更新功能、更新提醒
  • 多轮追问混乱 —— 多轮检索、加入历史对话信息
第 6 节：RAG 行业落地案例拆解 —— 产品设计思路复用 ▾

核心目标：拆解 3 个典型 RAG 落地案例，提炼产品设计共性

  • 案例 1：企业内部智能助手 —— 知识库轻量化、问答精准化
  • 案例 2：电商智能客服 —— 按品类/问题类型切分、多轮检索
  • 案例 3：政务问答机器人 —— 知识权威化、答案结构化
第 7 节：实战演练 —— 从 0 到 1 设计一个 RAG 产品 ▾

核心目标：整合前 6 节知识，完成简易 RAG 产品的设计 + 实操

实战场景：公司行政制度智能助手（RAG 版）

实战步骤

  • 步骤 1：需求分析与场景适配
  • 步骤 2：知识库设计（文档上传、切分、标签设置）
  • 步骤 3：检索策略设计（混合检索）
  • 步骤 4：撰写提示词模板
  • 步骤 5：落地实操（用免费工具搭建并测试）
  • 步骤 6：输出产品设计方案
第 8 节：答疑 + RAG 产品未来趋势 ▾
  • 学员问题答疑（聚焦产品设计、落地实操、提示词优化）
  • 未来趋势：多模态 RAG、RAG 与 Agent 结合、轻量化 RAG

Chapter 02 · Core Knowledge

各章节核心知识（产品视角，避技术）

四块核心知识，帮你建立 RAG 产品的完整认知体系。

2.1

场景适配

RAG 的产品适配与非适配场景

✓

适配场景

满足 1+2 即可

  • 有固定的知识库（文档/表格/数据库）
  • 需要基于知识库的精准问答
  • 知识需要频繁更新
  • 专业领域问答（如企业制度、医疗、法律）
✗

非适配场景

不建议使用 RAG

  • 无固定知识库，纯开放性创意生成（如文案、故事）
  • 不需要精准回答，只求灵感
  • 知识几乎不更新，且内容极少
2.2

4 大环节

RAG 4 大环节的产品影响点

核心环节 产品关注重点 对产品体验的影响
知识库构建 文档解析精度、知识切分粒度、更新方式、权限管理 解析不准→知识缺失；切分不合理→检索不准；更新复杂→知识过时
检索 检索精度、速度、结果数量、相似度阈值 精度低→答非所问；速度慢→交互卡顿；结果过多→答案冗长
提示词拼接 结构合理性、上下文筛选、指令清晰度 结构差→大模型理解错；指令模糊→幻觉/脱离上下文
生成 答案格式、来源标注、兜底逻辑 格式不规范→体验差；无来源→可信度低；无兜底→用户迷茫
2.3

知识库设计

知识库产品设计的核心要点

①

文档支持

优先支持企业常用格式（docx、pdf、excel、txt），如需解析图片/表格，需提前和技术确认

②

知识切分

建议按"问题粒度"切分，比按章节切分检索更准

③

知识标签

为每个知识块加标签（如"假期制度 - 年假"），支持按标签检索

④

更新方式

轻量场景设计手动更新，高频更新场景设计自动更新

⑤

权限管理

企业内部场景需设计按部门/角色权限

2.4

兜底逻辑

RAG 产品的核心兜底逻辑（产品必须设计）

①

检索不到匹配内容

"暂无相关知识解答，建议咨询 XX 部门"，不允许大模型编造

②

知识库未更新

"当前知识库暂未更新该内容，敬请期待"

③

超出问答边界

"本助手仅解答 XX 问题，该问题请咨询 XX 部门"

④

答案存疑

"当前相关知识内容暂不明确，建议查阅 XX 文档或咨询 XX 部门"

Chapter 03 · Prompt Engineering

RAG 提示词撰写 ——产品经理专属方法

掌握 RAG 提示词的核心差异、4 大原则和 3 套可直接复用的通用模板。

3.1

核心差异

RAG 提示词与普通提示词的本质区别

普通

普通大模型提示词

指令 + 用户问题（如"请简洁回答，年假怎么申请？"）

RAG

RAG 提示词

指令 + 检索上下文 + 用户问题（核心：绑定上下文，限制大模型仅基于上下文回答）

PM

产品经理核心要求

让提示词"标准化、模板化"，便于开发落地，且能快速优化

3.2

4 大原则

产品经理撰写 RAG 提示词的核心原则

①

边界明确

必须加"仅基于以下提供的上下文回答，不编造、不添加额外信息"

②

格式指定

明确答案格式（如"分点回答，条理清晰"），适配产品体验

③

上下文聚焦

"请从上下文中提取关键信息回答，不要冗长，无关信息无需提及"

④

适配场景

根据产品场景调整语气（企业助手：正式简洁；客服：口语亲切）

3.3

通用模板

3 套可直接复用的 RAG 提示词模板

基础版 适用于单轮精准问答，如行政智能助手
指令：请你作为XX（产品角色，如公司行政智能助手），仅基于以下提供的上下文回答用户问题，严格遵循以下要求： 1. 仅使用上下文的信息，不编造、不添加任何额外信息； 2. 答案分点回答，条理清晰，语言简洁正式，贴合企业内部沟通场景； 3. 若上下文无相关问题的解答，直接回复"暂无相关知识解答，建议咨询公司行政/财务部门"； 4. 提取上下文关键信息即可，无需冗长表述。 上下文：{检索到的知识库内容，由系统自动填充} 用户问题：{用户的实际问题}
多轮版 适用于需要上下文的多轮对话
指令：请你作为XX（产品角色），基于以下提供的上下文和历史对话信息，回答用户的最新问题，严格遵循以下要求： 1. 仅使用上下文和历史对话中的信息，不编造、不添加任何额外信息； 2. 答案分点回答，条理清晰，语言简洁正式； 3. 若上下文和历史对话中无相关问题的解答，直接回复"暂无相关知识解答"； 4. 提取关键信息即可，无需冗长表述。 历史对话：{历史对话内容，由系统自动填充} 上下文：{检索到的知识库内容，由系统自动填充} 用户最新问题：{用户的实际问题}
客服版 适用于电商客服场景
指令：请你作为电商客服智能助手，基于以下提供的上下文回答用户问题，严格遵循以下要求： 1. 仅使用上下文的信息，不编造、不添加任何额外信息； 2. 答案口语化、亲切友好，符合客服沟通风格； 3. 若上下文无相关问题的解答，直接回复"抱歉，暂时没有相关信息，建议您联系在线客服"； 4. 回答简洁明了，重点突出。 上下文：{检索到的知识库内容，由系统自动填充} 用户问题：{用户的实际问题}
